## 前后端契约

- **命令**（前端 `invoke` → Rust `#[tauri::command]`）：
  - `start_batch(tasks, options)` 启动一批（立即返回，进度走事件）
  - `stop_batch()` 取消并 `taskkill` 所有 robocopy
  - `scan_size(path)` 预扫描 `{ bytes, files }`
  - `verify_now(source, destination, createSubfolder, full)` 手动校验
  - `load_state()` / `save_state(state)` 持久化（应用配置目录 `state.json`）
  - `open_log_dir()` 资源管理器打开日志目录
- **事件**（Rust `emit` → 前端 `listen`）：
  - `progress`：单任务状态/百分比/速度/ETA/消息
  - `log`：单行实时日志（带任务 id）
  - `batch-done`：整批汇总（成功/失败/是否取消）
- **命名约定**：Rust 参数 snake_case，JS 传 camelCase，Tauri 自动转换；
  跨边界的结构体用 `#[serde(rename_all = "camelCase")]` 与 TS 对齐。

## 并发模型

`start_batch` 在一个协调线程里建立一个共享队列 `VecDeque<TaskInput>`，并启动
`concurrency` 个 worker 线程；每个 worker 循环 `pop_front` 取任务执行，队列空或被取消则退出。
这样**天然把并行度限制为 N**，无需信号量。全部 worker `join` 后 emit `batch-done`。

取消：`stop_batch` 置 `cancel` 原子标志，并对记录在案的 robocopy PID 执行
`taskkill /T /F`，确保即使卡在单个大文件也能立刻中断。

## 进度估算（与语言环境无关）

robocopy 的本地化输出（中文系统会输出“新文件”等）不适合按文字解析。采用：

1. **预扫描**源目录得到 `total_files` / `total_bytes`。
2. robocopy 加 `/BYTES /NP /NDL /TEE`，使每个被处理文件输出一行；行内含**纯数字字节数**
   和**反斜杠路径**。据此累加 `processed_files` / `copied_bytes`。
3. `percent` 以文件数为准（即使有跳过也能走到 100%）；`speed = copied_bytes / 耗时`；
   `eta = 剩余字节 / speed`。进度事件按 ~150ms 节流。

> 这是工程上的折中：robocopy 不提供精确的整体进度，文件数 + 字节累加足够给出可信的进度条与 ETA。
> 同时仍把完整输出写入每任务日志文件，作为权威记录。

## 校验设计

`verify.rs` 不解析 robocopy 输出，而是**自己遍历源目录**，逐个文件核对目标：

- `quick`：存在性 + 文件大小
- `full`：在 quick 基础上比对 sha256（分块读取，64KB 缓冲）

返回 `checked / missing / sizeMismatch / hashMismatch` 与抽样问题清单，完全与系统语言无关。

## robocopy 参数映射

| 选项 | 含义 |
| --- | --- |
| `/MIR` 或 `/E`、`/S` | 镜像 / 含空目录 / 不含空目录 |
| `/COPY:DAT /DCOPY:DAT` | 复制数据+属性+时间戳 |
| `/FFT /XJ` | 宽松时间比较、跳过 junction，避免无谓重拷与死循环 |
| `/R:n /W:n` | 重试次数 / 间隔 |
| `/MT:n` | 多线程 |
| `/BYTES /NP /NDL /TEE /LOG+:file` | 精确字节、无百分号刷屏、不列目录、同时输出到控制台与日志 |
| `/Z /J /XO /L` | 断点续传 / 大文件无缓冲 / 目标较新不覆盖 / 预演 |
| `/XD /XF` | 排除目录 / 文件 |
