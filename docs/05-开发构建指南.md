# 05 · 开发与构建指南

当前主线是 [`app/`](../app) 下的 Tauri 2 项目。前端可单独构建验证；要真正运行/打包桌面应用，
需要一次性安装 Rust 工具链与 Windows 构建依赖。

## 0. 前置依赖一览

| 依赖 | 说明 | 是否已具备（迁移时本机） |
| --- | --- | --- |
| Node.js + npm | 前端开发与构建 | ✅ Node 24 / npm 11 |
| Rust（rustup） | 后端编译 | ❌ 需安装 |
| MSVC C++ Build Tools | Rust MSVC 目标链接所需 | ❌ 需安装 |
| WebView2 Runtime | Tauri 渲染层（Win10/11 多已自带） | 多数已有 |
| winget / git | 安装工具 / 版本管理 | ✅ |

## 1. 一次性安装工具链

在 **PowerShell** 里执行（联网）：

```powershell
# Rust（含 cargo、rustc）
winget install --id Rustlang.Rustup -e

# Visual Studio C++ 生成工具（MSVC 链接器 + Windows SDK）
winget install --id Microsoft.VisualStudio.2022.BuildTools -e `
  --override "--quiet --add Microsoft.VisualStudio.Workload.VCTools --add Microsoft.VisualStudio.Component.Windows11SDK.22621"

# WebView2（如系统未自带）
winget install --id Microsoft.EdgeWebView2Runtime -e
```

装完**重开终端**，确认：

```bash
rustc --version
cargo --version
```

> 也可参考官方前置说明：https://tauri.app/start/prerequisites/

## 2. 开发调试

```bash
cd app
npm install          # 首次：装前端依赖
npm run tauri dev    # 启动应用（前端热重载；首次会编译 Rust，较慢）
```

## 3. 构建发布

```bash
cd app
npm run tauri build
```

产物在 `app/src-tauri/target/release/`：

- 可执行文件 `Smart Copy Tool.exe`
- `bundle/` 下的安装包（NSIS `.exe` / MSI，取决于环境）

`tauri.conf.json` 里 `bundle.targets` 当前为 `"all"`，可按需改为 `["nsis"]` 等只产出特定包。

## 4. 仅验证前端（无需 Rust）

适合改完界面快速自查（CI 也可用）：

```bash
cd app
npm install
npm run build        # tsc 严格类型检查 + vite 打包到 dist/
```

> 本项目 `tsconfig` 开了 `strict` / `noUnusedLocals` / `noUnusedParameters`，
> 有未用变量或类型错误会直接构建失败。

## 5. 常见问题

- **`error: linker 'link.exe' not found`**：MSVC Build Tools 没装好或没重开终端。
  重装第 1 步的 BuildTools，并新开终端。
- **`cargo` 找不到**：rustup 装完未刷新 PATH，重开终端或重登录。
- **白屏 / 起不来**：缺 WebView2 Runtime，装第 1 步的 EdgeWebView2Runtime。
- **首次 `tauri dev` 很慢**：Rust 依赖首次编译耗时正常，之后增量很快。
- **杀进程/权限**：复制需要对源读、对目标写的权限；网络盘需保持连接。

## 6. 数据与日志位置

- 持久化状态（任务、参数、预设）：应用配置目录下 `state.json`。
- 每个任务的 robocopy 日志：应用配置目录下 `logs/<任务id>.log`。
- 应用配置目录通常为 `C:\Users\<你>\AppData\Roaming\com.zhengzhangbiao-jk.smart-copy\`，
  可用界面「打开日志目录」直接定位。

## 7. 代码导航

后端从 `src-tauri/src/lib.rs` 的命令入手；复制逻辑在 `copy.rs`，校验在 `verify.rs`，
共享结构在 `model.rs`。前端从 `src/App.tsx` 入手，命令/事件封装在 `src/api.ts`。
架构细节见 [03 · 架构设计](./03-架构设计.md)。
