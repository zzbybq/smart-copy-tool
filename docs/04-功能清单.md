# 04 · 功能清单与使用说明

## 界面区域

1. **顶栏**：标题 + 预设栏（选择 / 载入 / 另存为 / 删除）。
2. **源 / 目标**：手输或“选择…”弹出系统目录选择；勾选“在目标下创建源同名文件夹”，点「添加任务」。
3. **全局参数**：并行任务数、线程/任务、重试、间隔、若干开关、校验模式、排除目录/文件。
4. **操作行**：开始复制 / 停止，右侧显示状态提示。
5. **任务表**：每行一个任务，含进度、速度、剩余时间、状态、改/删。
6. **日志面板**：实时输出，可清空、可打开日志目录。

## 六大功能

### 1. 任务级参数 + 并行
- 每个任务可单独选 **增量 / 镜像**，单独勾选 **同名子目录**（表格内直接改）。
- 全局“并行任务数 N”：同时运行 N 个任务。跨不同物理磁盘/网络目标时提速明显；
  同一块机械盘上并行过多反而更慢，建议 1–2。

### 2. 实时进度 + 速度 + ETA
- 进度条按“已处理文件数 / 总文件数”推进，并显示 `已处理/总数`。
- 速度按已复制字节 / 耗时估算；剩余时间按剩余字节 / 速度估算。
- 数值为估算值；权威结果以日志与退出码为准。

### 3. 任务编辑 + 拖动排序
- **拖动**行左侧把手可调整执行顺序。
- 点行内「改」弹窗编辑源/目标/模式/子目录；「删」删除该任务。

### 4. 完成通知
- 整批结束后弹**系统通知**并播放**提示音**；窗口保持打开，状态行给出汇总。

### 5. 校验模式
- `关闭`：只做复制。
- `快速(大小)`：复制后遍历源，核对目标文件是否存在且大小一致。
- `完整(哈希)`：再比对 sha256，最可靠但更慢。
- 校验不通过时，任务标红，日志列出抽样问题文件（缺失 / 大小不一致 / 内容不一致）。

### 6. 命名预设
- 「另存为」把当前**全部任务 + 全局参数**存成一个命名预设（同名覆盖）。
- 下拉选择后「载入」一键还原；「删除」移除预设。
- 任务、参数、预设都会自动持久化到应用配置目录的 `state.json`。

## 复制策略说明

- **增量**：复制新增/变化的文件，**不删除**目标里多出来的内容（安全，推荐迁移用）。
- **镜像**：让目标完全跟随源，会**删除**目标里源所没有的文件；启动前有二次确认。
- **目标较新不覆盖**（`/XO`）：避免新机器上较新的文件被旧文件覆盖。
- **同名子目录**：源 `D:\ai_work\desktop` + 目标 `D:\work_files` → `D:\work_files\desktop`。

## 建议用法

1. 配好一对源/目标，点「添加任务」；重复添加多对。
2. 先把“校验”设为关闭、可勾“预演(不实际复制)”，点开始，看任务表与日志是否符合预期。
3. 确认无误后取消预演，正式复制；大迁移可开“完整(哈希)”校验过夜跑。
4. 走无线很慢时优先换网线或同一高速 Wi-Fi；大量小文件可把线程调到 32，机械盘吃不消则降到 8。
5. 不确定就别用“镜像”——它会删目标里多出来的文件。

## 关于“不会丢文件”

工具通过 `robocopy` 的跳过/重试/断点/日志，加上可选的复制后校验，尽量提高可靠性并可验证。
但它**不能突破**网络、磁盘、权限、文件被占用等物理限制。退出码 0–7 通常表示成功或仅有可接受差异，
8 及以上需查看日志。
