# 01 · 背景与需求

## 起因

需要把大量文件（工作目录、素材、项目）在磁盘之间、或迁移到新电脑。用 Windows 资源管理器
直接拷有几个老问题：

1. 中途遇到一个被占用/超长路径/权限文件就**弹窗卡住**，要人盯着点。
2. 出错往往**整批中断**，没有自动重试。
3. 没有日志，事后无法确认“到底有没有拷全”。

Windows 自带的 `robocopy` 正好解决这些：多线程、自动重试、可断点续传、支持超长路径、
出错继续、可写日志。于是工具的定位是：**给 `robocopy` 套一个好用的图形界面**。

## 第一版（PowerShell）暴露的问题

最初用 PowerShell + WinForms 写了个单脚本版本，跑起来后发现三个实际痛点（也是本轮迭代的起点）：

1. **复制粒度不符合直觉**：源 `D:\ai_work\desktop` 拷到 `D:\work_files`，
   `robocopy` 默认把 desktop 的**内容**铺进 `work_files`，而不是生成 `work_files\desktop`。
2. **多任务只跑一个就停**：脚本用 robocopy 进程的 `Exited` 事件串接下一个任务，
   该回调在后台线程触发，在 PowerShell WinForms 里极不稳定，常常静默中断。
3. **跑完窗口像“消失了”**：其实是上一条崩溃的连带表现。

> 这三点的修复过程见 [02 · 演进历程](./02-演进历程.md)。

## 明确的需求清单

在修复上述问题后，进一步确定了 6 项增强需求，构成了当前 Tauri 版本的功能基线：

1. **任务级参数 + 并行**：每个任务单独存“增量/镜像、是否同名子目录”；支持“同时跑 N 个任务”。
2. **实时进度**：进度条 + 速度(MB/s) + 剩余时间(ETA)。
3. **任务列表可编辑 + 拖动排序**：弹窗编辑，拖拽调整顺序。
4. **完成后通知**：系统通知 + 提示音。
5. **校验模式**：复制后核对，给“没丢文件”一个可验证结论。
6. **命名预设**：整套任务+参数存为预设，下拉切换。

## 约束与边界

- **平台**：Windows 专用（依赖 `robocopy` / `taskkill` / `explorer`）。
- **不突破物理限制**：网络带宽、磁盘速度、文件占用、权限等问题工具无法绕过，只能提高可靠性与可观测性。
- **目标产物**：一个原生、小体积、可双击运行的桌面应用。
